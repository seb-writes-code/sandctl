# Design: TypeScript Template Management Commands

**Date:** 2026-02-23
**Status:** Approved
**Scope:** Port Go template subcommands to sandctl-ts

## Problem

The Go CLI has 5 template management subcommands (`add`, `list`, `show`, `edit`, `remove`) that are not yet implemented in the TypeScript rewrite. The TS version only has template *execution* during `sandctl new -T`. Users cannot create, browse, or manage templates from the TS CLI.

## Design Decisions

- **Full Go parity:** config.yaml metadata support, editor auto-opening, interactive confirmation
- **Flat file organization:** Each subcommand in its own file under `src/commands/` (matches existing patterns)
- **DI pattern:** All commands use interface-based dependency injection for testability

## Template Store Enhancement

### New types (`src/template/types.ts`)

```typescript
interface TemplateConfig {
  template: string;        // normalized name (e.g., "ghost")
  original_name: string;   // user-provided name (e.g., "Ghost")
  created_at: string;      // ISO 8601 timestamp
  timeout?: string;        // optional duration (e.g., "10m")
}
```

### New TemplateStore methods (`src/template/store.ts`)

| Method | Signature | Purpose |
|--------|-----------|---------|
| `add` | `(name: string) => Promise<TemplateConfig>` | Create directory + config.yaml + init.sh template |
| `get` | `(name: string) => Promise<TemplateConfig>` | Read and parse config.yaml |
| `list` | `() => Promise<TemplateConfig[]>` | All templates sorted by created_at |
| `remove` | `(name: string) => Promise<void>` | Delete template directory recursively |
| `exists` | `(name: string) => Promise<boolean>` | Check if template directory exists |
| `getInitScriptPath` | `(name: string) => Promise<string>` | Filesystem path to init.sh |

### New error type

`TemplateAlreadyExistsError` — thrown by `add()` when template directory already exists.

### Storage format (matches Go)

```
~/.sandctl/templates/<normalized-name>/
├── config.yaml  (0o600 permissions)
└── init.sh      (0o700 permissions)
```

## Commands

### Parent: `sandctl template` (`src/commands/template.ts`)

Registers all 5 subcommands. Shows help when run alone.

### `sandctl template add <name>` (`src/commands/template-add.ts`)

1. Validate name is not empty
2. Call `store.add(name)` → creates directory, config.yaml, init.sh with template content
3. Detect editor (EDITOR → VISUAL → vim → vi → nano)
4. Open init.sh in editor via `child_process.spawn({ stdio: 'inherit' })`
5. Print: "Template '<name>' created. Use `sandctl new -T <name>` to use it."

Errors: empty name → exit 1, already exists → suggest `template edit`, no editor → show file path.

### `sandctl template list` (`src/commands/template-list.ts`)

1. Call `store.list()`
2. Output table: `NAME | CREATED` (luxon for date formatting)
3. Empty state: "No templates configured. Use `sandctl template add <name>` to create one."

### `sandctl template show <name>` (`src/commands/template-show.ts`)

1. Call `store.getInitScript(name)`
2. Print raw init.sh content to stdout

Errors: template not found.

### `sandctl template edit <name>` (`src/commands/template-edit.ts`)

1. Validate template exists
2. Detect editor (same as `add`)
3. Open init.sh in editor

Errors: template not found, no editor found.

### `sandctl template remove <name>` (`src/commands/template-remove.ts`)

Flags: `--force/-f` (skip confirmation)

1. Validate template exists
2. Without `--force`: prompt "Delete template '<name>'? [y/N]" using `@inquirer/prompts`
3. Call `store.remove(name)`

Errors: template not found.

## Init Script Template

Generated by `store.add()`, matches Go version:

```bash
#!/bin/bash
# Init script for template: <Name>
# This script runs on the sandbox VM after creation.
#
# Available environment variables:
#   SANDCTL_TEMPLATE_NAME       - Original template name
#   SANDCTL_TEMPLATE_NORMALIZED - Normalized template name (lowercase)

set -e

echo "Template '<Name>' initialized successfully"
```
